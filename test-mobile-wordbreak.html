移动端英文单词断词测试

修复前的问题

"这里是东周的个人博客，专注于哲学(Philosophy)、物理(Physics)、心理学(Psychology)的交叉探索。"

问题描述：英文单词在括号内，移动端不会自动断词，导致文本溢出。

CSS状态：word-break: normal, overflow-wrap: normal

.home-info .entry-content p {
  word-break: normal;
  overflow-wrap: normal;
}

修复后的效果

"这里是东周的个人博客，专注于哲学(Philosophy)、物理(Physics)、心理学(Psychology)的交叉探索。"

解决方案：使用CSS强制断词，确保英文单词在移动端正确换行。

CSS修复：

/* 基础断词 */
.home-info .entry-content p {
  word-wrap: break-word;
  overflow-wrap: break-word;
  word-break: break-word;
}

/* 移动端特别处理 */
@media (max-width: 768px) {
  .home-info .entry-content p {
    word-break: break-all;
    overflow-wrap: break-word;
  }

  /* 特别处理第一段 */
  .home-info .entry-content p:first-of-type {
    word-break: break-all;
    overflow-wrap: anywhere;
  }
}

验证方法

  1. 在浏览器中打开此页面
  2. 使用开发者工具切换到移动端视图（375px宽度）
  3. 观察"修复前"部分：英文单词可能溢出
  4. 观察"修复后"部分：英文单词应该正确换行
  5. 实际测试：访问 https://dondrub3.com/

CSS属性说明：

  • word-break: break-all：在任意字符处断词，最激进
  • word-break: break-word：在单词内断词，较美观
  • overflow-wrap: break-word：在需要时断词
  • overflow-wrap: anywhere：在任何位置断词

备选方案

如果CSS修复不够理想，可以考虑：

  1. 修改文本：在英文单词中添加连字符或空格
  2. 使用软连字符：Philosophy → Philo­sophy
  3. 调整布局：减少字体大小或增加容器宽度
  4. 响应式设计：在不同屏幕尺寸使用不同文本
