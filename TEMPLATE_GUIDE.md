# 3P 博客文章模板使用指南

## 快速开始

### 根据文章类型选择模板

**交易3P分析**（投资、交易、市场相关）：
```bash
# 复制交易模板
cp archetypes/posts-trading.md content/posts/2026-03-10-文章标题.md
```

**决策3P分析**（战略、决策、思维相关）：
```bash
# 复制决策模板
cp archetypes/posts-decision.md content/posts/2026-03-10-文章标题.md
```

### 使用 Hugo 命令创建

```bash
# 使用默认模板
hugo new content posts/YYYY-MM-DD-文章标题.md

# 示例
hugo new content posts/2026-03-10-example-3p-analysis.md
```

## Front Matter 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `title` | ✅ | 文章标题 | `"当油价冲破100美元：..."` |
| `date` | ✅ | 发布日期 | `2026-03-09T14:00:00+08:00` |
| `draft` | ✅ | 是否为草稿 | `false` (发布) / `true` (草稿) |
| `tags` | ✅ | 文章标签 | `["油价", "能源危机", "3P分析"]` |
| `categories` | ✅ | 文章分类 | `["交易3P分析"]` 或 `["决策3P分析"]` |
| `summary` | 可选 | 文章摘要 | 用于列表页显示 |
| `author` | 可选 | 作者 | `"东周"` |

### 分类选择

| 分类 | 适用场景 | 结语对象 | 模板文件 |
|------|----------|----------|----------|
| `交易3P分析` | 投资、交易、市场分析 | 交易者 | `posts-trading.md` |
| `决策3P分析` | 战略、决策、思维方法 | 我们/读者 | `posts-decision.md` |
| `哲学` | 纯哲学思考 | 思考者 | `posts-decision.md` |
| `心理学` | 纯心理学分析 | 学习者 | `posts-decision.md` |

**选择原则：**
- 文章主要讨论市场、价格、投资机会 → **交易3P分析**
- 文章主要讨论思维、决策、认知 → **决策3P分析**

## 文章结构规范

### 1. Front Matter（元数据）

```yaml
---
title: "文章标题"
date: 2026-03-10T16:00:00+08:00
draft: false
tags: ["3P分析", "..."]
categories: ["交易3P分析"]
summary: "..."
---
```

**⚠️ 注意：**
- `---` 必须独占一行
- 字段缩进使用空格（不要用 Tab）
- 字符串值用双引号包裹
- 列表用方括号 `[]`

### 2. 引言（Quote）

```markdown
> 文章核心观点，一句话总结
```

- 必须紧跟 front matter 后面
- 用 `>` 引用格式
- 简洁有力，突出核心洞察

### 3. 导语

```markdown
## 导语

背景介绍...
```

- 说明热点事件或话题背景
- 引出为什么要用3P框架分析

### 4. 三个维度分析

```markdown
## 一、Physics：物理规律与系统分析

### 1.1 子标题
内容...

## 二、Psychology：心理机制与行为分析

### 2.1 子标题
内容...

## 三、Philosophy：哲学思考与价值反思

### 3.1 子标题
内容...
```

- 每个维度必须有
- 使用二级标题 `##`
- 子标题使用三级标题 `###`

### 5. 结语（必须有行动建议）

```markdown
## 结语：给读者的行动建议

### 核心洞察回顾

- **Physics**：物理层面的关键发现...
- **Psychology**：心理层面的核心洞察...
- **Philosophy**：哲学层面的深度思考...

### 具体行动建议

**如果你是投资者：**
- 建议1：...
- 建议2：...
- 建议3：...

**如果你是决策者：**
- 建议1：...
- 建议2：...
- 建议3：...

**如果你是普通读者：**
- 建议1：...
- 建议2：...

### 最后的提醒

> 记住东周的3P原则：科学 ≠ 真理，模型有边界，保持谦卑与好奇。

在不确定的世界中，保持清醒的头脑，做好应对各种可能的准备。
```

**结语要求：**
- ✅ 回顾三个维度的核心洞察
- ✅ 给不同读者群体具体建议（投资者/决策者/普通读者）
- ✅ 用 quote 强调核心原则
- ✅ 结尾有鼓舞或提醒

### 6. 文末声明

```markdown
*本文写于2026年3月10日，基于公开信息分析，不构成投资建议。*
*3P框架：Physics（物理/规律）、Psychology（心理/行为）、Philosophy（哲学/价值观）*
```

## 常见错误

### ❌ 错误1：正文重复标题

```markdown
---
title: "文章标题"
---

# 文章标题  ← 错误！front matter 已经有标题了

> 引言
```

### ✅ 正确

```markdown
---
title: "文章标题"
---

> 引言（直接从 quote 开始）
```

### ❌ 错误2：缺少 front matter

```markdown
# 文章标题

正文...
```

### ✅ 正确

```markdown
---
title: "文章标题"
date: 2026-03-10
---

> 引言
```

### ❌ 错误3：front matter 格式错误

```markdown
---
title: 文章标题    ← 错误！字符串必须用引号
date: 2026-03-10   ← 错误！需要完整格式
tags: 标签1        ← 错误！必须是列表格式
---
```

### ✅ 正确

```markdown
---
title: "文章标题"
date: 2026-03-10T16:00:00+08:00
tags: ["标签1", "标签2"]
---
```

## 发布流程

1. **创建文章**
   ```bash
   hugo new content posts/2026-03-10-article-title.md
   ```

2. **编辑内容**
   - 填写 front matter
   - 写入正文（不要重复标题）

3. **本地预览**
   ```bash
   hugo server -D
   ```

4. **构建站点**
   ```bash
   hugo --minify
   ```

5. **提交部署**
   ```bash
   git add -A
   git commit -m "Add new article: 文章标题"
   git push origin main
   ```

## 结语示例（参考）

### 交易3P分析 - 结语示例

```markdown
## 结语：交易者的行动指南

### 核心洞察回顾

- **Physics**：油价破百不是投机泡沫，而是物理供给约束的硬着陆
- **Psychology**：恐惧与贪婪的钟摆正在摆动，市场情绪接近极端
- **Philosophy**：能源依赖的深层困境需要我们重新定义"进步"

### 具体交易建议

**仓位管理：**
- 能源板块仓位不超过总仓位的20%，避免过度集中
- 采用分批建仓策略，不要一次性满仓
- 设置止损位，油价回落到85美元以下考虑减仓

**风险管理：**
- 检查组合的相关性，避免多个能源相关资产同时暴露
- 准备对冲方案，如买入能源ETF的看跌期权
- 保持现金比例，等待更好的入场时机

**心理建设：**
- 警惕FOMO情绪，不要因为别人赚钱就盲目跟进
- 接受错过，市场上永远有机会
- 坚持交易计划，不要被短期波动打乱节奏

### 最后的提醒

> 记住东周的3P交易原则：科学 ≠ 真理，模型有边界，保持谦卑与好奇。在不确定的市场中，活着比赚钱更重要。

做好应对各种可能的准备，我们下期见。
```

### 决策3P分析 - 结语示例

```markdown
## 结语：我们的行动指南

### 核心洞察回顾

- **Physics**：AI Agent从云端走向终端是计算范式迁移的必然
- **Psychology**：认知卸载带来效率提升，但也带来依赖风险
- **Philosophy**：工具理性不能压倒价值理性，技术应服务于人的全面发展

### 具体行动建议

**认知升级：**
- 理解端侧AI的技术原理和应用边界
- 关注AI Agent对产业链的重构影响
- 建立技术趋势与投资决策的连接思维

**决策优化：**
- 评估AI工具对你工作/生活的真实价值
- 有意识地选择哪些能力保留，哪些可以外包
- 建立"断开连接"的时间，保持独立思考能力

**生活实践：**
- 尝试使用AI Agent处理低价值重复任务
- 保留核心决策权，不让算法完全替代判断
- 定期反思：技术是否真的让生活更美好

### 最后的提醒

> 记住东周的3P原则：科学 ≠ 真理，模型有边界，保持谦卑与好奇。

在不确定的世界中，保持清醒的头脑，做好应对各种可能的准备。
```

### 不好的结语示例

```markdown
## 结语

总之，这是一个复杂的问题，涉及很多因素。我们需要关注动态，做出明智的决策。
```

**问题：**
- ❌ 没有回顾三个维度
- ❌ 没有具体行动建议
- ❌ 过于笼统，没有价值
- ❌ 没有区分读者群体
